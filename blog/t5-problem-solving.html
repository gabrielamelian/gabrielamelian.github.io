Problem Solving

The theme for this week's blog is finding solutions to problems, my favorite part of working on IT! I have always had this taste for puzzles and for finding solutions to the weirdest hardest problems. Since I started programming it obviously comes with a slightly different baggage and specially different tools that you can use to help you through the way.

In the following lines I have to reflect upon a time when I had felt blocked and (another) that I was able to overcome and resolve it in an elegant way. When learning to code you get blocked with silly things once every 10 minutes or so, the thing is to learn to get out of these issues easily. I will also go through some of the techniques we learned on the course and specify how I'm feeling about them.

Problem solving techniques and processes

We had this long list of different troubleshooting techniques to go through, but I don't want to bore you presenting all of them at once. I would like to mention the ones I found most useful mainly so I can remember them better.

Pseudocode is writing the skeleton of the program in a piece of paper without using code but just the logic. This is something I have been using a lot since before I started the course. It helps specially when you don't know exactly how to do something (like when you are just learning a programming language for example) but you can line up the steps of how the problem can be resolved. From there I usually go and try to fill everything out with what I know, and google what I still don't know. I then try and give it a first run, but at this stage I know something will surely fail. From there it is finding the problems and solving them one by one. A really helpful tool when working with Javascript is using Developer Tools in Chrome.

I am lucky to have a few friends that know programming and I can ask questions when I get stuck. They have to be good questions as in per try to find the solution yourself and if you don't find it then, and just then, articulate an intelligent way to get the answer you want. Otherwise you get sent to RTFM or more politely to "google" what you are looking for.

Javascript Array functions

I will explain with my own words a couple of array functions in Javascript. I have to say I banged my head against the wall for some time trying to understand these concepts using the recommended documentation MDN Mozilla Developer Network which is way too "academic" can we say (more like cryptic, but hey! To each their own). I did understand finally when I read the W3Schools documentation.

  • Map .map()

    • This method applies a given function to each element of an array and then returns a new array containing these results. The original array remains unchanged.

  • Filter .filter()

    • This method uses a given function to check through to each element of an array: if the result is true it returns the element, otherwise it discards it. From this method you also obtain a new array containing only the elements that passed the "check". The original array remains unchanged.

  • Reduce .reduce()

    • This method seems to be quite mathematic in the sense that it will go through each item in an array and apply a function which will reduce it to a single element. I could not see (or more accurately understand) one example that was not used to produce a mathematic result such as the sum of all elements. I found this really friendly article that explains it using loops, the problem is that the loops are already clean and clear, then why use reduce at all? The miseries of Javascript...
